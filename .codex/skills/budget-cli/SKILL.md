# budget-cli

Use this skill for work in the Python CLI budget app repository.

## When to Use
- Implementing or reviewing CSV-based transaction features.
- Adding or updating tests for budget calculations and filtering.
- Enforcing project rules from `AGENTS.md`.

## Required Practices
- Write tests before production code.
- Keep functions typed and short.
- Prefer small, reviewable changes.
- Run `pytest` and `radon cc` during validation.

## Suggested Review Flow
1. Identify the behavior change.
2. Write or update tests.
3. Implement the smallest change that passes.
4. Check complexity and function length.
5. Ask `qa_engineer` to review before commit.
